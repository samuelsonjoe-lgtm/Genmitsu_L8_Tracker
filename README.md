# Genmitsu L8 Tracker v0.9.0

A single-file web app for logging and organizing laser settings for a Genmitsu L8 20W diode laser engraver, so results are repeatable across sessions.

## Files to use

- `index.html` - standalone offline baseline. This is the preferred file to open locally because it has no `support.js`, x-dc, or Google Fonts dependency.
- `Genmitsu L8 Tracker.dc.html` - original Claude/Grok artifact export kept for reference and comparison.

## Why this exists

Getting clean, repeatable cuts and engraves out of a diode laser depends on a pile of interacting numbers - power, speed, passes, line interval, focus height, air assist, and more - that vary by material and thickness. Without a system, that knowledge lives in scattered notes (or nowhere), and every new sheet of material means re-guessing settings from scratch. This tool gives that data a permanent, searchable home.

## What it does

The app has eight views:

- **Log** - a quick-entry record of every job you run: material, thickness, job type (cut/engrave), power (with optional min/max ramp), speed, passes, line interval, air assist + pressure, overscan %, kerf offset, dither mode, focus height/material Z-offset, software used, settings file name, a 1-5 result rating, and free-form notes. Searchable, sortable, and filterable by material.
- **Library** - a curated set of "best known settings" per material/thickness, promoted from log entries once you've dialed something in. Manage mode preserves the searchable/sortable/taggable card reference and remains the legacy-safe default. Browse mode adds a material -> thickness -> operation tree with a selected profile detail, suggested Inventory matches, and related Projects. Its Material Test Wizard guides matrix, interval, cut, kerf, and manual tests into each material's test history, with an explicit option to apply winners to primary settings. Library materials can also retain optional proven production settings with machine/material condition, cut settings, signed LightBurn kerf reference, separate joint and Drawer Cabinet fit values, manual evidence, confidence/status, and preferred/superseded history. These records are reviewable evidence for one physical condition, not universal defaults; Designs may copy only explicitly checked, template-compatible values into its session-only form.
- **Test Grids** - a power x speed test matrix planner for dialing in a brand-new material: set ranges and step sizes, get a grid, then click each cell to record a rating/notes and mark winning combinations. New Grids automatically snapshot the active machine, normally Genmitsu L8 20W, so Joe does not need to select a machine for ordinary use; an explicit advanced override supports L8 40W and custom identities. The stored machine remains fixed when the global preference changes. Manage mode is the compatibility default and preserves the existing cards and matrix workflow. Browse mode adds a material -> operation -> Test Grid tree, search and operation filtering, selected-grid completion and best-cell summaries, and completed Library Material Test promotion history. Any cell can still be promoted straight to a new Library draft; optional structured Material Test recording and profile-local provenance deduplication remain in the matrix workflow. Recorded results can be exported as CSV. Older machine-less Grids remain unassigned until explicitly updated; Grid promotion and the Grid -> Material Test bridge use only the recorded Grid machine and never substitute the current preference. Machine identity records provenance; it does not prove a result is verified or production-safe.
- **Designs** - session-only offline SVG generators for freestanding QR/sign stands, hanging signs, dice trays, divider trays, open-top finger-jointed boxes with an optional loose flat lid, a sliding-lid finger box, a Drawer Cabinet with one to three equal drawers in each of one to three rows and columns (up to nine drawers), and compact Joint Fit Coupons. The Joint Fit Coupon's default Finger-edge clearance mode emits separate A/B panels that assemble at 90 degrees, using the same complementary finger geometry as the box generators across an explicit list of signed clearance values; small coupon-only negative values test interference fits, and optional blue score labels use deterministic vector glyphs without fonts. Its separate Wall-to-base tab clearance mode uses the selected four- or two-tab tray wall profile to emit two identical test walls and one multi-row base plate; the candidate rows are top-to-bottom in entered order, have no engraved labels, and cannot be promoted yet, so any physical winner must be recorded manually. Both modes are local fit-learning tools, not proof of a full tray, corner strength, glue strength, warping, cumulative assembly error, Divider retention, or production safety. Coupon results remain material- and kerf-specific and select a setting for the next full prototype rather than replacing final box testing. Finger Box and Drawer Cabinet joint fields accept small negative interference values down to -0.10 mm, but negative values require physical prototyping with the same material, thickness, focus, cut settings, and measured kerf; they do not guarantee a press fit. The Drawer Cabinet derives an open-front, closed-back five-panel shell and equal-size open-top drawers. Linked thickness is the default: one measured shell thickness applies to the shell, drawers, plain edge-glued shelves, and plain row-height glued partitions, producing one production SVG. When intentionally enabled, separate thickness uses shell thickness for the five finger-jointed shell panels and drawer/interior thickness for every drawer, shelf, and partition, producing separate Cabinet shell and Drawers, shelves, and partitions production SVGs. No direct finger-jointed pair uses unequal thicknesses. Partitions have no tabs, slots, finger joints, interlocking geometry, or placement score guides; measure, dry-fit, and glue them manually while confirming equal bay widths. Cabinet joints, drawer joints, lateral clearance, vertical clearance, and rear clearance remain independent experimental controls, with total drawer lateral clearance defaulting to 0.45 mm. Optional Shelf-placement guides add short blue score marks on the Left, Right, and Back interior faces at actual upper-shelf elevations; they run before red cuts and are positioning aids only, not structural support. Shelves still require edge glue, and the first physical cabinet must confirm the marked interior face and top/bottom orientation before relying on them. The sliding-lid box produces eight through-cut pieces: five body panels, a separate lid, and two identical laminated glue-on inner channel rails. The lid enters at the Front and slides toward the Back stop. Its form distinguishes usable cavity below the lid, full wall-to-wall interior, and outside body dimensions, with separate body joint, lid side, lid vertical, and Front insertion clearances. It supports an optional centered rectangular edge pull, optional blue rail-placement score guides for the corrected Back-flush rail footprint, optional assembly labels on the supported box faces, and an optional six-piece sliding-fit coupon; guides and labels share one blue score operation before red cuts, labels use deterministic vector glyphs with no font dependency and remain readable in sheet space, the marked face is intended to assemble inward, and a Right side panel becomes handed whenever either guides or labels are enabled. Labels may be omitted from parts without the required safe clearance while valid cut output remains exportable; verify visibility and face orientation during the first scrap assembly. Kerf is not included and remains a separate LightBurn/material-test concern. Rails and cabinet shelves require glue, and all body, drawer, sliding, and coupon fits require scrap testing and manual physical verification. The template selector is grouped by design type. Drawer Cabinet and Sliding-Lid Box keep common structural controls visible and use a native, closed-by-default Advanced disclosure for less-common fit and marking options; closing it never disables, resets, removes, or saves those draft values. This form organization does not change production SVGs or downloads. An optional session-only Production setting panel can review saved Library Cut records and copy only explicitly checked compatible measured-thickness or fit values into ordinary editable Design fields. Finger Box and Drawer Cabinet expose their supported fit mappings; Joint Fit Coupon, Sliding Lid Box, QR stand, Dice tray, and Divider tray accept measured thickness only, while Hanging sign shows no panel. Machine mismatches still allow measured thickness with a warning but block mechanical-fit values. Cut settings and signed LightBurn kerf remain reference-only, are never added to SVG geometry, and Designs never writes back to Library records. The selected source is not persisted. Designs are not saved and are excluded from JSON backups.
- **Gift Box** - a session-only G1 parametric keepsake-box generator with internal or external dimensions, exposed finger joints, lift-off or surface-mounted hinged lids, flush or front/side-overhanging lids, optional four-rail locating frame, blue measured hinge placement guides, optional front hardware marks, and an optional hinge placement coupon. Guides are drilling references rather than mortises or screw holes; measure stock and hardware, test on scrap, assign LightBurn layers, and keep an active fire watch. Run `index.html?selftest=gift-box` for the focused group.
- **Concealed Cleat Corner Prototype** - a session-only Designs template that produces one combined SVG for an experimental three-way visible-butt-seam, butt-and-cleat corner. It uses measured thickness, an outside Base/Wall-A leg, a shortened Wall B, laminated cleat strips, mandatory blue placement ticks, and outer-layer labels. It is glue-dependent, not self-squaring, not a strength test, and is not a Finger Box option.
- **Concealed Cleat Full-Box Prototype** - a separate session-only experimental Designs template for an open-top four-wall butt-and-cleat box. It uses outside width, depth, wall height, and measured thickness to create four laminated base-cleat stacks and four Wall-A/Wall-C-owned seam-cleat stacks. Blue four-corner guides register each exact installed footprint; green deterministic labels identify panels and outer cleat layers; red paths cut the parts. SVG layer colors do not set LightBurn operations, so assign both blue and green layers to Line mode after import. The Finished View is screen-only. This is glue-dependent, requires manual squaring and physical dry-fit, and does not establish clamp access, fit, strength, durability, or production safety.
- **Reference** - official Genmitsu L8 20W and 40W speed/power starting points from SainSmart's resource center, selected by the separate **Genmitsu L8 Reference profile** selector, plus focus, safety, maintenance, offline, and software notes pulled from the user manual. The 20W/40W tables are bundled Genmitsu L8 manufacturer starting points, not universal settings for another machine. The header shows the active owned workshop machine and provides quick switching; the full **Workshop machines** manager in Reference can add, edit, duplicate, archive, restore, and permanently delete only unused non-active machines. Each machine stores a friendly name, manufacturer, model, type, nominal power, advisory usable work area, and notes. The work area is advisory only - it does not resize designs, perform nesting, guarantee bed fit, control the machine, or change Reference settings - and you enter both dimensions together or leave both blank. New Grids, Material Tests, production settings, promoted evidence, and Project settings snapshot the active owned machine with an optional stable `machineId`; historical records are not backfilled, and Reference Copy to Library remains unowned with its existing attribution. M3 identity matching blocks machine-dependent promotion when IDs differ or only one side has an ID. Any reference setting can be copied into the Library with its existing Genmitsu Reference attribution and tuned from there.
- **Projects** - a searchable/taggable gallery of finished pieces with a saved settings snapshot, small offline photos, tags, notes, optional sale/cost/profit accounting, filtered accounting summaries, status filtering, accounting CSV export, and a copy-to-log action for repeating a past project. Manage mode is the legacy-safe default and preserves the sortable Project cards. Browse mode adds a status -> primary material -> Project tree with selected Project details, source-record context, suggested Inventory matches, and related finished batches. The optional Project Wizard has four steps: Material, Operation and baseline, Production goal, and Review. It selects verified on-hand Inventory, ranks Best Known, Experimental, Saved setting, or Unverified Library/Test evidence, copies eligible baselines unchanged, and opens the normal Project form for review and saving; it never deducts Inventory automatically. Phase 1 goals change the explanation and testing guidance only; they do not automatically change numeric laser settings. The accounting report remains above both modes and continues to follow the current Project filters. Projects can also be opened as drafts from Pricing estimates.
- **Inventory** - a lightweight manual tracker for raw materials on hand and simple finished-item batches, with low-stock badges, estimated raw-material value, remaining finished counts, starter-material helpers, canonical material aliases, and CSV exports. It includes Manage cards and a collapsible Browse mode with search, in-stock filtering, stock details, suggested Library matches and Material Tests, related Projects, and draft actions. Manage is the safe default for legacy data; Finished Batches remain separate and nothing auto-deducts stock.
- **Pricing** - a compact cost/profit calculator for estimating revenue, material and consumable costs, machine time, labor, fees, margin, break-even price, target sale prices, saved rate/fee defaults, printable summaries, CSV export, and a copyable summary before committing to a price.

## Current storage model

The standalone `index.html` app saves live data in the browser's `localStorage` under `genmitsu-l8-tracker-v1`. That means data is tied to the browser/profile that opened the app. Use Export regularly for portable JSON backups until a future desktop build writes to a real app data file.

JSON backups include app/version/backup-format metadata alongside Log entries, Library profiles and their nested Material Tests/proven production settings/evidence, Test Grids, Projects, Project photos, Inventory records including raw-material aliases, unit preference, saved sort preferences, the Library, Test Grids, Projects, and Inventory Manage/Browse preferences, Library match helper inputs, selected Genmitsu L8 Reference profile, the root `machines` collection and `activeMachineId`, the compatibility `machineIdentity` mirror, Pricing draft, and Pricing defaults. New production-critical record snapshots may carry their source `machineId`; unknown IDs are retained and no historical IDs are fabricated. Each stored machine keeps its stable ID and owned profile fields; the active machine's name and advisory usable work-area width/height are mirrored into `machineIdentity` for existing consumers. A legacy payload with no own `machines` field migrates once to the fixed `legacy-workshop-machine` record, including a blank legacy identity; an explicitly present empty or malformed `machines` field does not fabricate that record. **Merge import** unions explicit incoming machines by ID while local records and the current active selection win; it does not synthesize a machine from an imported legacy mirror. **Replace import** restores the imported root model and active selection and performs the same legacy migration when `machines` is absent. Machines can be managed in Reference and switched from the header, while `machineProfile` remains separate and historical records are unchanged. Production-setting dimensions and fit values are stored canonically in millimeters and are not rewritten when the display unit changes. Session-only browser selections and expanded nodes, search text, tag/status filters, active tab, and open modal state are intentionally not backed up. Legacy backups without the new metadata remain supported. Backups declaring a newer storage schema are refused rather than silently imported into an older data format. Missing or invalid `gridViewMode` values safely default to Manage.

If the saved `localStorage` value is malformed, the app opens with temporary empty data and blocks normal saving so the damaged value is not silently replaced. Use the recovery controls before continuing.

## Fields tracked

Log entries, Library profiles, Test Grids, and Projects track material, thickness, job type, power (min/max), speed, passes, line interval, air assist (on/off + pressure), overscan %, kerf offset, dither mode (for photo engraves), focus height / material Z-offset, software (e.g. LightBurn), settings file reference, result rating, and notes where applicable. Optional Library production settings additionally separate measured material condition, machine, cut settings, signed LightBurn through-cut kerf offset, finger-joint clearance, Drawer Cabinet total lateral clearance, verification status, and evidence. Kerf remains reference data for the red through-cut layer; it is not baked into SVG geometry or applied to blue score/engrave layers. Inventory tracks raw material stock counts/costs and finished-batch quantities manually, and its raw-material names/aliases can help keep material names consistent; it does not auto-deduct from Projects.
