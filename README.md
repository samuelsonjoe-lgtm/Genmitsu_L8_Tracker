# Genmitsu L8 Tracker

A single-file web app for logging and organizing laser settings for a Genmitsu L8 20W diode laser engraver, so results are repeatable across sessions.

## Files to use

- `index.html` - standalone offline baseline. This is the preferred file to open locally because it has no `support.js`, x-dc, or Google Fonts dependency.
- `Genmitsu L8 Tracker.dc.html` - original Claude/Grok artifact export kept for reference and comparison.

## Why this exists

Getting clean, repeatable cuts and engraves out of a diode laser depends on a pile of interacting numbers - power, speed, passes, line interval, focus height, air assist, and more - that vary by material and thickness. Without a system, that knowledge lives in scattered notes (or nowhere), and every new sheet of material means re-guessing settings from scratch. This tool gives that data a permanent, searchable home.

## What it does

The app has eight views:

- **Log** - a quick-entry record of every job you run: material, thickness, job type (cut/engrave), power (with optional min/max ramp), speed, passes, line interval, air assist + pressure, overscan %, kerf offset, dither mode, focus height/material Z-offset, software used, settings file name, a 1-5 result rating, and free-form notes. Searchable, sortable, and filterable by material.
- **Library** - a curated set of "best known settings" per material/thickness, promoted from log entries once you've dialed something in. Manage mode preserves the searchable/sortable/taggable card reference and remains the legacy-safe default. Browse mode adds a material -> thickness -> operation tree with a selected profile detail, suggested Inventory matches, and related Projects. Its Material Test Wizard guides matrix, interval, cut, kerf, and manual tests into each material's test history, with an explicit option to apply winners to primary settings. Library materials can also retain optional proven production settings with machine/material condition, cut settings, signed LightBurn kerf reference, separate joint and Drawer Cabinet fit values, manual evidence, confidence/status, and preferred/superseded history. These records are reviewable evidence for one physical condition, not universal defaults; Designs may copy only explicitly checked, template-compatible values into its session-only form.
- **Test Grids** - a power x speed test matrix planner for dialing in a brand-new material: set ranges and step sizes, get a grid, then click each cell to record a rating/notes and mark winning combinations. New Grids automatically snapshot the active machine, normally Genmitsu L8 20W, so Joe does not need to select a machine for ordinary use; an explicit advanced override supports L8 40W and custom identities. The stored machine remains fixed when the global preference changes. Manage mode is the compatibility default and preserves the existing cards and matrix workflow. Browse mode adds a material -> operation -> Test Grid tree, search and operation filtering, selected-grid completion and best-cell summaries, and completed Library Material Test promotion history. Any cell can still be promoted straight to a new Library draft; optional structured Material Test recording and profile-local provenance deduplication remain in the matrix workflow. Recorded results can be exported as CSV. Older machine-less Grids remain unassigned until explicitly updated; Grid promotion and the Grid -> Material Test bridge use only the recorded Grid machine and never substitute the current preference. Machine identity records provenance; it does not prove a result is verified or production-safe.
- **Designs** - session-only offline SVG generators for freestanding QR/sign stands, hanging signs, dice trays, divider trays, open-top finger-jointed boxes with an optional loose flat lid, a sliding-lid finger box, a Drawer Cabinet with one to three equal drawers in each of one to three rows and columns (up to nine drawers), and compact Joint Fit Coupons. The Joint Fit Coupon's default Finger-edge clearance mode emits separate A/B panels that assemble at 90 degrees, using the same complementary finger geometry as the box generators across an explicit list of signed clearance values; small coupon-only negative values test interference fits, and optional blue score labels use deterministic vector glyphs without fonts. Its separate Wall-to-base tab clearance mode uses the selected four- or two-tab tray wall profile to emit two identical test walls and one multi-row base plate; the candidate rows are top-to-bottom in entered order, have no engraved labels, and cannot be promoted yet, so any physical winner must be recorded manually. Both modes are local fit-learning tools, not proof of a full tray, corner strength, glue strength, warping, cumulative assembly error, Divider retention, or production safety. Coupon results remain material- and kerf-specific and select a setting for the next full prototype rather than replacing final box testing. Finger Box and Drawer Cabinet joint fields accept small negative interference values down to -0.10 mm, but negative values require physical prototyping with the same material, thickness, focus, cut settings, and measured kerf; they do not guarantee a press fit. The Drawer Cabinet derives an open-front, closed-back five-panel shell and equal-size open-top drawers. Linked thickness is the default: one measured shell thickness applies to the shell, drawers, plain edge-glued shelves, and plain row-height glued partitions, producing one production SVG. When intentionally enabled, separate thickness uses shell thickness for the five finger-jointed shell panels and drawer/interior thickness for every drawer, shelf, and partition, producing separate Cabinet shell and Drawers, shelves, and partitions production SVGs. No direct finger-jointed pair uses unequal thicknesses. Partitions have no tabs, slots, finger joints, interlocking geometry, or placement score guides; measure, dry-fit, and glue them manually while confirming equal bay widths. Cabinet joints, drawer joints, lateral clearance, vertical clearance, and rear clearance remain independent experimental controls, with total drawer lateral clearance defaulting to 0.45 mm. Optional Shelf-placement guides add short blue score marks on the Left, Right, and Back interior faces at actual upper-shelf elevations; they run before red cuts and are positioning aids only, not structural support. Shelves still require edge glue, and the first physical cabinet must confirm the marked interior face and top/bottom orientation before relying on them. The sliding-lid box produces eight through-cut pieces: five body panels, a separate lid, and two identical laminated glue-on inner channel rails. The lid enters at the Front and slides toward the Back stop. Its form distinguishes usable cavity below the lid, full wall-to-wall interior, and outside body dimensions, with separate body joint, lid side, lid vertical, and Front insertion clearances. It supports an optional centered rectangular edge pull, optional blue rail-placement score guides for the corrected Back-flush rail footprint, optional assembly labels on the supported box faces, and an optional six-piece sliding-fit coupon; guides and labels share one blue score operation before red cuts, labels use deterministic vector glyphs with no font dependency and remain readable in sheet space, the marked face is intended to assemble inward, and a Right side panel becomes handed whenever either guides or labels are enabled. Labels may be omitted from parts without the required safe clearance while valid cut output remains exportable; verify visibility and face orientation during the first scrap assembly. Kerf is not included and remains a separate LightBurn/material-test concern. Rails and cabinet shelves require glue, and all body, drawer, sliding, and coupon fits require scrap testing and manual physical verification. An optional session-only Production setting panel can review saved Library Cut records and copy only explicitly checked compatible measured-thickness or fit values into ordinary editable Design fields. Finger Box and Drawer Cabinet expose their supported fit mappings; Joint Fit Coupon, Sliding Lid Box, QR stand, Dice tray, and Divider tray accept measured thickness only, while Hanging sign shows no panel. Machine mismatches still allow measured thickness with a warning but block mechanical-fit values. Cut settings and signed LightBurn kerf remain reference-only, are never added to SVG geometry, and Designs never writes back to Library records. The selected source is not persisted. Designs are not saved and are excluded from JSON backups.
- **Reference** - official Genmitsu L8 20W and 40W speed/power starting points from SainSmart's resource center, selected by machine profile, plus focus, safety, maintenance, offline, and software notes pulled from the user manual. Any reference setting can be copied into the Library and tuned from there.
- **Projects** - a searchable/taggable gallery of finished pieces with a saved settings snapshot, small offline photos, tags, notes, optional sale/cost/profit accounting, filtered accounting summaries, status filtering, accounting CSV export, and a copy-to-log action for repeating a past project. Manage mode is the legacy-safe default and preserves the sortable Project cards. Browse mode adds a status -> primary material -> Project tree with selected Project details, source-record context, suggested Inventory matches, and related finished batches. The optional Project Wizard has four steps: Material, Operation and baseline, Production goal, and Review. It selects verified on-hand Inventory, ranks Best Known, Experimental, Saved setting, or Unverified Library/Test evidence, copies eligible baselines unchanged, and opens the normal Project form for review and saving; it never deducts Inventory automatically. Phase 1 goals change the explanation and testing guidance only; they do not automatically change numeric laser settings. The accounting report remains above both modes and continues to follow the current Project filters. Projects can also be opened as drafts from Pricing estimates.
- **Inventory** - a lightweight manual tracker for raw materials on hand and simple finished-item batches, with low-stock badges, estimated raw-material value, remaining finished counts, starter-material helpers, canonical material aliases, and CSV exports. It includes Manage cards and a collapsible Browse mode with search, in-stock filtering, stock details, suggested Library matches and Material Tests, related Projects, and draft actions. Manage is the safe default for legacy data; Finished Batches remain separate and nothing auto-deducts stock.
- **Pricing** - a compact cost/profit calculator for estimating revenue, material and consumable costs, machine time, labor, fees, margin, break-even price, target sale prices, saved rate/fee defaults, printable summaries, CSV export, and a copyable summary before committing to a price.

Other features:

- **Finger Box faux dovetail engraving** is an optional session-only Corner decoration that adds dovetail-style taper marks only to the blue score layer at the actual vertical finger positions. It is decorative only: the structural joint remains a finger joint, red cut geometry, fit, and strength are unchanged, and the blue score layer should run before red cuts. Engraved faces should face outward; verify appearance on scrap before production. It is not a true structural dovetail.
- **Drawer Cabinet custom-by-row layouts** retain Uniform grid mode and add a session-only Custom by row mode: each one-to-three-row cabinet may use one, two, or three equal-width drawers independently per row, such as top-to-bottom `1 / 3` or `2 / 1`, up to nine drawers. The row with the most drawers establishes cabinet width from the entered drawer inside width; rows with fewer drawers are proportionally wider, while all rows remain the same height. There are no unequal widths within one row, spans, cubbies, or interlocking partitions. Shelves and row-height partitions remain plain manually positioned glued pieces with no partition score guides. Linked thickness still exports one production file; separate shell/interior thickness still exports the two required files. The bounded Finished Front View shows the row-specific drawer widths, but physical dry-fitting and verification remain required.
- **Dice Tray underside cover plate** is an optional separate full-size plain component, manually edge-aligned beneath the structural base to conceal wall-slot openings and tab ends. It is cosmetic only, does not strengthen or replace the structural joint, and adds one piece, one material thickness, material use, and cut time. Confirm wall tabs are flush, clean both glue surfaces, and align the plate carefully before clamping; confirm concealment, alignment, and flatness on the first real tray. The Dice Tray Finished View intentionally does not show this production cover geometry in this phase.
- Toggle between imperial and metric units, with stored thickness/focus/Z-offset values converted when switching.
- Export/import your data and portable app preferences, including Library production settings and evidence, Test Grids, Projects, and Inventory Manage/Browse preferences, as a JSON backup file with merge or replace import modes. Old backups without production settings remain compatible.
- Storage failures are caught without overwriting unreadable browser data. The recovery panel can download the damaged raw data, import a valid backup, or explicitly start fresh. Photo-heavy JSON exports also show an approximate size warning before download.
- Material suggestions include saved materials, official SainSmart reference materials, and raw Inventory material names with stock status when available.
- Inventory raw materials can store comma-separated aliases so Amazon-style listing titles or old shorthand can match a clean canonical material name without blocking free typing.
- Material safety warnings flag obvious risky/unknown terms such as PVC, vinyl, chlorinated plastics, unknown plastics, fiberglass, and carbon fiber without blocking saves. Inventory Browse suppresses laser-action drafts for clear unsafe or mystery cases until composition is verified.
- Laser, Project, and Pricing fields include short hover/focus help for beginner-friendly terminology.
- Library includes a closest-saved-setting helper for matching material and thickness.
- Keyboard shortcuts: `Ctrl+N` creates a new item in the active view, `Ctrl+S` saves the open form, `Esc` closes modals or exits grid detail, and `/` focuses the active search field.
- Delete actions show a short undo window before the removal becomes final.
- Duplicate Log entries and Library profiles for quick tweak-and-resave runs.
- Print a simplified Library cheat sheet for shop use. Printing from Browse still uses the complete filtered Library card summary rather than only the selected profile.
- Pricing calculator drafts/defaults persist locally between sessions and are included in the main JSON backup/export.
- **Create project from estimate** on the Pricing tab opens a prefilled Project draft (user must still click Save project).
- Project accounting can copy a matched raw Inventory unit cost into the current Project as an explicit one-time action.
- Project accounting can track optional material line items and copy their total into the Project material cost field when you choose.
- Project photos are resized/compressed before storage and ride inside Project records; image-heavy backups can get larger.
- **Accounting report** on the Projects tab summarizes filtered project revenue, cost, profit, margin, quantity, and status counts.
- **Export accounting CSV** on the Projects tab exports one row per project with accounting inputs and calculated totals, including a material line count and material line total.
- **Inventory CSV exports** save filtered raw-material stock and finished-batch counts as plain CSV files.
- All data persists automatically in the browser (`localStorage`) between sessions. Pricing results update immediately while browser writes are briefly debounced to avoid serializing the full tracker on every keystroke.
- Log, Projects, Library, Test Grids, and Inventory searches update only their result areas, preserving search focus and avoiding unnecessary full-tab rebuilds.

## Current storage model

The standalone `index.html` app saves live data in the browser's `localStorage` under `genmitsu-l8-tracker-v1`. That means data is tied to the browser/profile that opened the app. Use Export regularly for portable JSON backups until a future desktop build writes to a real app data file.

JSON backups include Log entries, Library profiles and their nested Material Tests/proven production settings/evidence, Test Grids, Projects, Project photos, Inventory records including raw-material aliases, unit preference, saved sort preferences, the Library, Test Grids, Projects, and Inventory Manage/Browse preferences, Library match helper inputs, selected machine profile, Pricing draft, and Pricing defaults. Production-setting dimensions and fit values are stored canonically in millimeters and are not rewritten when the display unit changes. Session-only browser selections and expanded nodes, search text, tag/status filters, active tab, and open modal state are intentionally not backed up. Missing or invalid `gridViewMode` values safely default to Manage.

If the saved `localStorage` value is malformed, the app opens with temporary empty data and blocks normal saving so the damaged value is not silently replaced. Use the recovery controls before continuing.

## Fields tracked

Log entries, Library profiles, Test Grids, and Projects track material, thickness, job type, power (min/max), speed, passes, line interval, air assist (on/off + pressure), overscan %, kerf offset, dither mode (for photo engraves), focus height / material Z-offset, software (e.g. LightBurn), settings file reference, result rating, and notes where applicable. Optional Library production settings additionally separate measured material condition, machine, cut settings, signed LightBurn through-cut kerf offset, finger-joint clearance, Drawer Cabinet total lateral clearance, verification status, and evidence. Kerf remains reference data for the red through-cut layer; it is not baked into SVG geometry or applied to blue score/engrave layers. Inventory tracks raw material stock counts/costs and finished-batch quantities manually, and its raw-material names/aliases can help keep material names consistent; it does not auto-deduct from Projects.

Finger-jointed box Designs includes a session-only **Finished View** alongside the default **Cut Layout**. New views use the existing `buildBoxModel()` / `buildBoxDesignResult()` dimensions, show an open-top assembled box with Front, Back, Left, and Right orientation, and show a loose lid as a separate flat panel labeled `Loose lid`; no hinge, rail, retention, or sliding mechanism is implied. Sliding-lid finger box Designs adds session-only **Finished Closed** and **Finished Open** views beside Cut Layout: Front is the lid-entry end, Back contains the stop, and the lid rides on two inner rails. All Finished Views are screen-only, do not mutate the draft, are not persisted or included in backups, and downloads use the applicable production Cut Layout SVG. They do not prove smooth travel, fit, strength, kerf accuracy, glue quality, or production safety. Drawer Cabinet has a bounded session-only **Finished Front View** that shows its equal-size rows by columns drawer grid, horizontal shelves, and vertical partitions without consuming production SVG bytes. Tray choices are four-tab or two-tab wall-to-base profiles: wall-bottom tabs enter through-slots in the base, adjacent walls are butt joints, and dry fitting followed by glue is expected. This is not corner finger joinery; fit clearance remains a starting point that requires physical testing on scrap or with a future fit coupon. A post-audit cosmetic cleanup displays `1 divider` when `dividerCount` is 1 and `N dividers` for plural counts; it does not change production geometry, SVG, downloads, storage, or schemas. Its model-supplied divider positions and base-slot relationships are screen-only and do not prove retention, fit, strength, kerf accuracy, glue quality, or safety; no cross-divider system or true finger-jointed corners are implemented. Downloads remain the unchanged production Cut Layout SVG. Dice and Divider Tray production SVG remains derived from the structured model with its prior anonymous red-cut contract byte-for-byte, with no storage or schema migration. Dry-fit and physically test real material before production. The Designs geometry fixture group has 945 assertions, and the current isolated Edge `?selftest=all` run reports 1737 assertions at 0 failures.

Promotion offers Create, Update, or Evidence-only actions and saves only after review. New Test Grids created through the current UI automatically record the active machine; that stored identity remains fixed if the global preference changes. Old Grids without a machine remain machine-less until explicitly assigned, and only those machine-less Grids remain generally limited to Evidence-only or non-machine-dependent promotion. Machine identity records provenance; it does not prove a result is verified or production-safe.
