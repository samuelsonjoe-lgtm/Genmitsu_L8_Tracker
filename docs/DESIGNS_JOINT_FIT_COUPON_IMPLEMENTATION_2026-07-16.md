## Implemented feature

Added a session-only Designs template:

```text
Joint Fit Coupon
```

The default output tests:

```text
0.06, 0.04, 0.02, 0.00, -0.02, -0.04 mm
```

Each value produces two separately serialized physical panels:

```text
joint-coupon-p01-a
joint-coupon-p01-b
joint-coupon-p02-a
joint-coupon-p02-b
...
```

Every pair:

- shares one nominal pattern generated by `buildFingerPattern()`;
- uses one patterned top edge and three plain edges;
- uses measured material thickness as the recess depth;
- emits Coupon A with phase `true`;
- emits Coupon B with phase `false`;
- preserves normalized input order;
- emits A before B;
- is intended to assemble with the panel planes at 90 degrees.

This is the same local finger-joint geometry path used by Finger Box and Drawer Cabinet. No second finger-joint algorithm was introduced.

## Fixed-decimal signed-clearance parser

Added:

```javascript
parseJointCouponClearances()
```

Parsing rules:

- comma and/or line-break separators;
- optional leading plus or minus sign;
- fixed decimal syntax only;
- one or two decimal places required;
- exponent notation rejected;
- more than two decimal places rejected;
- blank tokens rejected;
- malformed tokens rejected;
- values converted to integer hundredths of a millimeter;
- `-0.00` normalized to `0.00`;
- duplicate integer-hundredths values rejected;
- normalized order preserved;
- minimum 3 values;
- maximum 8 values;
- accepted range `-0.10` through `+0.30 mm`, inclusive;
- normalized display text always uses exactly two decimals.

Each accepted entry has:

```javascript
{
  hundredths,
  value,
  text
}
```

Generated sequences and cumulative floating-point stepping are not used.

## Signed-clearance safety

The coupon model uses the existing required-web convention:

```text
required web = max(0.5 mm, material thickness × 0.25)
```

It validates every signed test value symmetrically:

```text
actual segment width - abs(clearance) >= required web
```

This protects the feature that becomes narrow:

- positive clearance narrows fingers;
- negative clearance narrows recesses.

The coupon grip depth must satisfy:

```text
grip depth >= max(4 × material thickness, 16 mm)
```

Existing Finger Box, Sliding Lid Box, Drawer Cabinet shell, and Drawer joint-clearance validation was not changed. Those templates continue rejecting negative joint clearance.

## Unverified physical assumptions

No plywood was cut during this implementation pass.

Software validation confirms geometry consistency, but does not prove:

- that `-0.02` or `-0.04 mm` is physically safe for a particular sheet;
- that a negative value will produce a reusable press fit;
- that char, focus, grain, humidity, or sheet compression will not alter fit;
- that a successful one-edge coupon guarantees a square or glue-free full box.

The user must:

- use the same measured sheet and laser setup intended for the project;
- assemble coupon mates at 90 degrees;
- test from positive relief toward zero and negative interference;
- stop before crushing, splitting, or permanent lock;
- validate the selected clearance in a full prototype.
