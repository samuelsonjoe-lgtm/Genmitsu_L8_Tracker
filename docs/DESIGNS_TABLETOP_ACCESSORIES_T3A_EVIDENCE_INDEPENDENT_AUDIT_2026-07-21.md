## 14. Regression found ‚Äî required correction before commit

**`runTabletopStorageFitCouponFixtures` regressed from 23/0 to 21/2.** Root cause, confirmed by reading both assertions directly:

```
add('outer-shell evidence scope and unproven mechanism are explicit',
  /outer shell/i.test(result.metrics.outerShellEvidenceScope)
  && /Unproven/.test(result.metrics.mechanismEvidenceStage)
  && !productionEvidenceKinds.includes('tabletop-storage-fit-coupon'), true, result.metrics);

add('Project handoff keeps the distinct mechanism facts in notes without a schema change',
  project?.name==='Tabletop Storage Fit Coupon'
  && /five retrofit pieces/i.test(project.notes)
  && /masking-tape pull tabs/i.test(project.notes)
  && /deferred/i.test(project.notes), true, project);
```

Both assertions were written during the **prior** (T3A design) phase to prove that evidence recording was *not yet* implemented ‚Äî one literally asserts `productionEvidenceKinds` does **not** contain `'tabletop-storage-fit-coupon'`; the other requires the Project note text to contain the word "deferred." This evidence-implementation phase correctly (and necessarily) added the kind to the allowlist and correctly (and appropriately) changed the note wording to *"Record a dedicated T3A physical result before any exact Storage Fit Coupon-proven promotion. Cork fit and the complete tray remain unproven."* ‚Äî which no longer contains "deferred." **The implementation never went back to update these two now-obsolete assertions**, so the group now fails 2/23 for a reason that reflects correctly-changed behavior, not a functional defect in the new geometry/production code. This is exactly the kind of gap a "focused-group-only" pass misses, and exactly why the audit prompt insisted on full-suite revalidation.

**This must be corrected before commit** ‚Äî every prior phase in this project has held a strict 0-failure bar across all registered fixture groups, and shipping with two known-red assertions (even though caused by stale expectations, not a real defect) would break that discipline and could mask a future genuine regression in the same group. The fix is small: update the first assertion to expect the kind to be present, and update the second to check for the new wording (or a more durable marker) instead of "deferred."
